# Premium UI Copy Guide

本文件用作統一平台前台文案方向，目標是令整體語氣更沉穩、簡潔、具品牌感，避免出現帶有 AI、OCR 或技術工具感的字眼。

## 文案方向

- 偏向內部行政平台語氣
- 以「整理」「帶入」「同步」「跟進」「摘要」等字眼取代技術式描述
- 盡量避免在前台頁面直接出現 AI、OCR、Batch、Line 等字眼
- 技術名詞可保留於開發說明，不建議直接顯示於前台

## 建議替換

| 原字眼 | 建議字眼 |
|---|---|
| 按附件自動讀取並填寫 | 整理附件內容 |
| 自動讀取內容並填入相關欄位 | 整理附件內容並帶入主要欄位 |
| 初版會按圖片內容自動讀取英文 / 中文資料 | 系統會按附件內容整理常見資料 |
| 請先上傳附件，再進行自動讀取 | 請先上傳附件，再開始整理 |
| 自動讀取失敗，請檢查網絡或稍後再試 | 內容整理未完成，請稍後再試或手動補回 |
| 儲存 Batch + Line | 儲存批次及明細 |
| 匯出 Lines CSV | 匯出明細 CSV |
| Batch / Line 結構 | 批次及明細結構 |
| 貨品報稅資料上傳 | 貨品報稅資料整理 |
| 本地記錄 | 案件記錄 |
| 即時核對 | 核對摘要 |
| 本地摘要預覽 | 一般模組摘要 |

## 補充建議

- 內容提示可多用「建議」「可先」「完成後」等柔和用語
- 按鈕盡量用簡潔動詞，例如：
  - 整理附件內容
  - 儲存案件記錄
  - 送出後續流程
  - 匯出明細 CSV
- 若頁面需要展示資料層次，建議以前台語氣顯示為：
  - 批次主表
  - 明細主表
  - 費用批次
  - 費用明細
